========================================
VALIDATIERAPPORT REZZERV v01.09.10
========================================

Release-type:
- Kleine backend-release
- Read-only preview-endpoint voor originele bonbestanden

Doel van deze release:
- De originele bronfile van een receipt veilig read-only beschikbaar maken voor latere UI-preview
- Geen parserwijziging, geen voorraadwijziging, geen Kassa-flowwijziging

Versie:
- Vorige versie: 01.09.09
- Nieuwe versie: 01.09.10

Wat is gewijzigd:
- Nieuw endpoint toegevoegd: GET /api/receipts/{receiptTableId}/preview
- Endpoint zoekt via receipt_table -> raw_receipt -> storage_path het originele bestand op
- Endpoint serveert het bestand read-only met inline content-disposition
- Ondersteunt in validatie ten minste PDF en PNG/JPG-mimetypes
- Veilige padcontrole toegevoegd: bronbestand moet binnen RECEIPT_STORAGE_ROOT liggen
- Nette foutafhandeling toegevoegd:
  - 404 bij onbekende bon
  - 404 bij ontbrekend origineel bestand
  - 403 als bestand buiten toegestane opslag ligt

Wat is NIET gewijzigd:
- Geen parserlogica aangepast
- Geen receipt_tables- of receipt_table_lines-logica aangepast
- Geen writes naar inventory_events
- Geen UI-wijziging
- Geen redesign
- Geen routerwijziging buiten nieuw preview-endpoint
- Geen Kassa-flow aangepast

Uitgevoerde validaties:
1. Python compile-check backend geslaagd
   - backend/app/main.py
   - backend/app/services/receipt_service.py
2. Health-check geslaagd
   - GET /api/health -> 200 {"status":"ok"}
3. PDF import + preview geslaagd
   - POST /api/receipts/import -> 201
   - GET /api/receipts/{id}/preview -> 200 application/pdf, inline
4. Afbeelding import + preview geslaagd
   - POST /api/receipts/import -> 201
   - GET /api/receipts/{id}/preview -> 200 image/png, inline
5. Ontbrekend bronbestand correct afgehandeld
   - GET /api/receipts/{id}/preview -> 404 {"detail":"Originele bon ontbreekt"}
6. Controle inventory-events
   - inventory_events voor test: 5
   - inventory_events na test: 5
   - dus geen voorraadwrites door deze release

Niet uitgevoerd / niet volledig afgetekend:
- Geen docker compose build --no-cache in deze omgeving
- Geen volledige browserregressierun in deze omgeving
- Geen UI-validatie, omdat dit bewust geen UI-release is

Risiconiveau:
- Laag tot middel
- Laag omdat het endpoint read-only is en bestaande parser/voorraadlogica ongemoeid laat
- Middel omdat een fout in bestandspaden of headers preview kan breken, maar niet de kernflow hoort te beschadigen

Eindoordeel:
- GO als backend-release voor read-only preview-ontsluiting
- Volgende logische stap: aparte UI-release voor Bon-preview card op Kassa-detail
