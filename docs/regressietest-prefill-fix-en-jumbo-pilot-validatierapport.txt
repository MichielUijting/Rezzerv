Rezzerv v01.07.10 — validatierapport

Doel van deze release
- regressietest in Admin robuust maken voor store import geheugen/prefill
- eerste gecontroleerde stap zetten voor ondersteuning van een tweede winkel

Uitgevoerde wijzigingen
1. Backend / dev fixture reset
- reset_dev_tables leegt nu ook:
  - purchase_import_lines
  - purchase_import_batches
  - household_store_connections
  - store_import_memory
  - inventory_events
- gevolg: regressietests starten weer vanuit een schone winkelimport-situatie en bouwen geen oude Lidl-prefills mee

2. Tweede winkelprovider toegevoegd
- nieuwe actieve mock provider toegevoegd: Jumbo
- provider seed is uitgebreid met jumbo
- mock batchmetadata toegevoegd:
  - aankoopdatum: 10-03-2026
  - winkel: Jumbo, Marktplein 8, Utrecht
- mock regels toegevoegd voor Jumbo

3. Admin browser-regressietest verbeterd
- Lidl-scenario gebruikt nu een schone fixture zonder oude memory-restanten
- Lidl verwerk-scenario kan omgaan met bestaande of automatische prefills via ensureStoreLineReadyForProcessing
- nieuw scenario toegevoegd:
  - Jumbo-flow opent kassabon na koppelen en aankopen ophalen

Validatie uitgevoerd
- Python compileall op backend/app: geslaagd
- frontend productiebuild via directe Vite entrypoint: geslaagd

Opmerking
- Volledige API-eindtest met FastAPI TestClient is in deze container niet uitgevoerd omdat projectafhankelijkheden zoals SQLAlchemy hier niet globaal beschikbaar zijn buiten de projectruntime.
- Deze release is daarom gevalideerd op broncode, buildniveau en regressierunnerlogica.

Niet gewijzigd
- geen brede redesign van Winkels
- geen live externe koppeling voor Jumbo
- geen wijziging van voorraadmodel
- geen wijziging van de werkende Lidl-flow buiten testrobustheid
