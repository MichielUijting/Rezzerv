Rezzerv v01.07.09 — validatierapport Release B winkelimport provider-neutrale UI

Aanleiding
- Opvolgende release na v01.07.08
- Doel: tweede stap uit het winkelimport-pad uitvoeren
- Extra eis: de eerder toegevoegde Lidl-regressietest in Admin moet behouden blijven

Uitgevoerde wijzigingen
- Backend wijzigingen uit de eerdere provider-neutrale stap zijn samengebracht met de basis van v01.07.08
- Store provider seedlogica is generiek gemaakt via STORE_PROVIDER_DEFINITIONS
- Mock pull-flow leest nu providergebonden datasets en batchmetadata
- Batch endpoint levert nu store_provider_name, purchase_date, store_name en store_label terug
- Inventory-event note gebruikt nu een generieke provider-notatie in plaats van Lidl-hardcoding
- StoresPage rendert providerkaarten op basis van /api/store-providers in plaats van vaste Lidl-selectie
- Batchheader gebruikt dynamische provider- en winkelmetadata in plaats van vaste Lidl-tekst in de componentlogica
- Admin browser regressietest voor de Lidl-flow is behouden in de opvolgende versie

Niet gewijzigd
- Geen tweede winkelprovider toegevoegd
- Geen brede UX-herbouw buiten het Winkels-scherm
- Geen wijziging van het voorraadmodel
- Geen wijziging van de bestaande Lidl-verwerkingsflow

Validatie
- Python syntax-check geslaagd: backend/app/main.py
- Frontend productiebuild geslaagd met Vite vanuit frontend/
- Handmatige codecontrole uitgevoerd op:
  - frontend/src/features/stores/StoresPage.jsx
  - frontend/src/features/admin/lib/browserRegressionRunner.js
  - backend/app/main.py
