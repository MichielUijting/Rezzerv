Deze informatie geldt voor het gehele project!! Ik heb een visiedocument opgesteld met de naam Mosterd (bijlage) en heb zelf een Styleguide opgesteld die onderdeel uitmaakt van de configuratie en mee moet groeien met de applicatie zelf. Ik test de applicatieversies lokaal met een link naar Github. Ik wil van de releasecoördinator weten wanneer ik een release moet vastzetten in Github. Dat wil ik niet per patch doen maar per mijlpaal in de ontwikkeling. De status van de applicatie is dat ik nog niet een volledig technisch draaiende versie heb. Met name de front end is nog niet stabiel in mijn testomgeving. De backend, frontend en browser zit op mijn laptop maar moet wel een werkende omgeving simuleren en met beperkte ingrepen cloudready worden. Ik heb als voorbeeld in een eerder stadium een Minimal Viable Product opgesteld waarmee je kunt inloggen, terecht komt in een landingspagina met uitstappen naar functionaliteit. Daarvan werkt het scherm "Voorraad" heel basaal en heb ik vooral aandacht besteed aan het kunnen inlezen en omzetten van een kassabon naar een tabel. Die tabel kan vervolgens worden toegevoegd aan de voorraad van het huishouden. Ik heb het visiedocument Mosterd, een versie van de MVP als zip en de styleguide. Verder heb ik de laatste versie van Rezzerv als zip toegevoegd waarvanuit we doorontwikkelen naar een volwaardige app Rezzerv. In eerste instantie concentreren we ons op een laptop-versie van de App en later zullen we een styleguide voor een mobiele app toevoegen. Deze documenten moeten beschikbaar blijven voor dit project en de agents. Mijn lokale map is "C:\Users\Gebruiker\OneDrive\Scans\Documenten\GitHub\Rezzerv" en ik gebruik een zip-importer om een terug ontvangen zip van het team te plaatsen in de lokale map.

Mijlpaal 1 — “Technisch Werkend Fundament”

Doel: Alles draait stabiel lokaal als gesimuleerde cloudomgeving.
Acceptatiecriteria:
Backend start zonder handmatige fixes
Frontend stabiel in browser
Login → Landingspagina → Voorraad werkt end-to-end
Kassabon → Tabel → Voorraad toevoegen werkt reproduceerbaar
Geen console errors
Zip-import workflow werkt zonder breuk
Styleguide wordt consistent toegepast

📌 Dit is de eerste release waardige milestone
Tagvoorbeeld: v0.1-foundation
Nog niet publiceren, maar wél GitHub Release aanmaken.
