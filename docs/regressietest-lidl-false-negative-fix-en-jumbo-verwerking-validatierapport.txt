Rezzerv v01.07.11 — validatierapport

Scope van deze release
- Herstel van de broze Admin-regressietest voor Lidl-verwerking.
- Uitbreiding van de regressietest met Jumbo-verwerking.
- Generieke weergave van winkelherkomst in Artikeldetails → Historie op basis van providercontext.

Uitgevoerde wijzigingen
1. browserRegressionRunner.js
- Verwerktest voor Lidl minder broos gemaakt:
  - niet meer afhankelijk van exacte succesregel in de UI
  - controle op echte voorraadtoename via /api/dev/inventory-preview
  - controle op winkelimport-historie via /api/dev/article-history
  - detailscherm blijft nog wel openen ter extra live-check
- Nieuwe helper toegevoegd voor async pollen in regressietests.
- Nieuw scenario toegevoegd:
  - Jumbo-flow kan een regel koppelen en naar voorraad verwerken
- Jumbo-verwerktest gebruikt dezelfde generieke flow als Lidl.

2. ArticleHistoryTab.jsx
- Opmerking bij winkelimport is niet meer hardcoded op Lidl.
- Provider wordt nu uit de event-note gelezen:
  - provider=lidl -> Geïmporteerd via Lidl
  - provider=jumbo -> Geïmporteerd via Jumbo
  - fallback -> Geïmporteerd via winkel

Validatie uitgevoerd
- Python syntax-check backend/app/main.py: geslaagd
- Frontend build via Vite: geslaagd

Opmerking
- Deze release bouwt voort op v01.07.10.
- De backendverwerking voor providers was al generiek genoeg; de kern van deze release zit in testrobustheid, Jumbo-verwerktestdekking en generieke historie-weergave.
